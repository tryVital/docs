This quickstart guide walks you through the essential steps to integrate Junction into your application. By the end, you'll have a working app that creates users, connects wearable devices, and fetches health data.

<Note>
Need help? Join our Slack channel or email [support@junction.com](mailto:support@junction.com) for assistance.
</Note>

## Prerequisites

Before you begin, ensure you have:
- A Junction account ([Sign up here](https://app.junction.com))
- Node.js 14+ and Python 3+ installed
- Basic familiarity with REST APIs

## Get your API keys

<Steps>
<Step title="Create a Junction account">
Sign up for a Junction account in the [Dashboard](https://app.junction.com).
</Step>

<Step title="Create a team">
Create a team by hovering over your username at the bottom of the Dashboard sidebar.

<Info>
Teams are **regional resources**. Choose either `US` or `EU` based on your data residency requirements. This cannot be changed later. Learn more about [regions](/api-details/regions).
</Info>
</Step>

<Step title="Generate API keys">
Navigate to the configuration section of the Dashboard. For each region, you'll have access to two environments:

| Environment  | Description                                 | Base URL                    |
| ------------ | ------------------------------------------- | --------------------------- |
| `sandbox`    | Testing environment (up to 50 users)        | api.sandbox.us.junction.com |
| `production` | Live environment for real customers         | api.us.junction.com         |

Create a new **Sandbox API key** to get started.

<Warning>
Store your API key securely. Never commit API keys to version control or expose them in client-side code.
</Warning>
</Step>
</Steps>

## Run the Quickstart app

<Steps>
<Step title="Clone the repository">
Clone the [Quickstart repository](https://github.com/tryVital/quickstart) to your local machine.

<Tabs>
<Tab title="macOS/Linux">
```bash
git clone https://github.com/tryVital/quickstart.git
cd quickstart
```
</Tab>

<Tab title="Windows">
```bash
# Use symlinks for correct behavior on Windows
git clone -c core.symlinks=true https://github.com/tryVital/quickstart.git
cd quickstart
```
</Tab>
</Tabs>

</Step>

<Step title="Configure environment variables">
Create a `.env` file in the root directory with your API credentials:

```bash .env
VITAL_API_KEY=your_sandbox_api_key_here
VITAL_REGION=us  # or 'eu'
VITAL_ENV=sandbox
```

<Tip>
You can copy `.env.example` as a starting point if it exists in the repository.
</Tip>
</Step>

<Step title="Install and run the backend">
Set up the Python backend server:

```bash
# Install dependencies using Poetry
poetry install

# Start the backend server
cd backend/python
source ./start.sh
```

<Check>
The backend server should now be running on `http://localhost:8000`
</Check>
</Step>

<Step title="Install and run the frontend">
Open a new terminal window and set up the frontend:

```bash
# Navigate to frontend directory
cd quickstart/frontend
npm install

# Configure frontend environment
# Edit .env.local with the following:
# NEXT_PUBLIC_VITAL_API_KEY=your_api_key
# NEXT_PUBLIC_VITAL_ENV=sandbox
# NEXT_PUBLIC_VITAL_REGION=us

# Start the frontend
npm run dev
```

<Check>
Visit `http://localhost:3000` to see your running application.
</Check>
</Step>
</Steps>

## Create your first user

Users are the core entity in Junction. Each user represents an individual whose health data you want to access.

### Understanding user IDs

<Info>
Junction uses two ID types:
- **`client_user_id`**: Your internal user identifier
- **`user_id`**: Junction's unique identifier (returned after creation)

Store the `user_id` in your database for future API calls.
</Info>

<Warning>
Never use personally identifiable information (PII) like email addresses or phone numbers as the `client_user_id`. Use a UUID or internal database ID instead.
</Warning>

### Create a user via the UI

<Frame>
<img src="/img/quickstart.jpg" alt="Junction Quickstart interface showing user creation form with client_user_id input field" />
</Frame>

Enter a unique `client_user_id` and click **Create** to generate a new user.

### Create a user via API

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.sandbox.us.junction.com/v2/user/ \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --header 'x-vital-api-key: YOUR_API_KEY' \
  --data '{
    "client_user_id": "user_12345"
  }'
```

```javascript Node.js
import { VitalClient, VitalEnvironment } from '@tryvital/vital-node';

const client = new VitalClient({
  apiKey: process.env.VITAL_API_KEY,
  environment: VitalEnvironment.Sandbox,
});

const user = await client.user.create({
  clientUserId: "user_12345"
});

console.log(`Created user: ${user.user_id}`);
```

```python Python
from vital.client import Vital
from vital.environment import VitalEnvironment

client = Vital(
  api_key=os.environ["VITAL_API_KEY"],
  environment=VitalEnvironment.SANDBOX
)

user = client.user.create(client_user_id="user_12345")
print(f"Created user: {user.user_id}")
```

```java Java
import com.vital.api.Vital;
import com.vital.api.core.Environment;
import com.vital.api.resources.user.requests.UserCreateBody;

Vital vital = Vital.builder()
  .apiKey(System.getenv("VITAL_API_KEY"))
  .environment(Environment.SANDBOX)
  .build();

UserCreateBody request = UserCreateBody.builder()
  .clientUserId("user_12345")
  .build();

var user = vital.user().create(request);
System.out.println("Created user: " + user.getUserId());
```

```go Go
import (
  "context"
  "fmt"
  "os"

  vital "github.com/tryVital/vital-go"
  vitalclient "github.com/tryVital/vital-go/client"
  "github.com/tryVital/vital-go/option"
)

client := vitalclient.NewClient(
  option.WithApiKey(os.Getenv("VITAL_API_KEY")),
  option.WithBaseURL(vital.Environments.Sandbox),
)

user, err := client.User.Create(context.TODO(), &vital.UserCreateBody{
  ClientUserId: "user_12345",
})
if err != nil {
  return err
}
fmt.Printf("Created user: %s\n", user.UserId)
```

```swift Swift
let user = try await VitalClient.shared.user.create("user_12345")
print("Created user: \(user.userId)")
```

</CodeGroup>

## Connect a wearable device

Junction Link enables your users to securely connect their wearable devices and health apps. The connection process uses a two-step authentication flow:

1. **Backend**: Generate a short-lived `link_token`
2. **Frontend**: Use the token to launch Junction Link Widget

### Generate a link token

<Info>
Link tokens are single-use and expire after a short period. Generate a new token for each connection attempt.
</Info>

<CodeGroup>

```python Backend (Python)
from vital.client import Vital
from vital.environment import VitalEnvironment

client = Vital(
  api_key=os.environ["VITAL_API_KEY"],
  environment=VitalEnvironment.SANDBOX
)

# Generate link token for a specific user
link_token = client.link.token(user_id=user.user_id)
return link_token
```

```javascript Backend (Node.js)
import { VitalClient, VitalEnvironment } from '@tryvital/vital-node';

const client = new VitalClient({
  apiKey: process.env.VITAL_API_KEY,
  environment: VitalEnvironment.Sandbox,
});

// Generate link token for a specific user
const linkToken = await client.link.token(userId);
return linkToken;
```

</CodeGroup>

### Launch Junction Link Widget

Use the React SDK to integrate Junction Link into your frontend:

```javascript React
import { useCallback } from "react";
import { useVitalLink } from "@tryvital/vital-link";

export function ConnectDeviceButton({ userId }) {
  const onSuccess = useCallback((metadata) => {
    console.log("Device connected:", metadata);
    // Refresh your UI or trigger data sync
  }, []);

  const onExit = useCallback((metadata) => {
    console.log("User exited:", metadata);
  }, []);

  const onError = useCallback((error) => {
    console.error("Connection error:", error);
  }, []);

  const config = {
    onSuccess,
    onExit,
    onError,
    env: "sandbox",
    region: "us",
  };

  const { open, ready } = useVitalLink(config);

  const handleConnect = async () => {
    // Fetch link token from your backend
    const response = await fetch(`/api/link-token/${userId}`);
    const { link_token } = await response.json();

    // Open Junction Link Widget
    open(link_token);
  };

  return (
    <button
      onClick={handleConnect}
      disabled={!ready || !userId}
    >
      Connect Device
    </button>
  );
}
```

## Fetch health data

Once a device is connected, you can retrieve health data through the Junction API. Let's fetch sleep data as an example.

### Get sleep summary data

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://api.sandbox.us.junction.com/v2/summary/sleep/USER_ID?start_date=2024-01-01&end_date=2024-01-31' \
  --header 'Accept: application/json' \
  --header 'x-vital-api-key: YOUR_API_KEY'
```

```python Python
from vital.client import Vital
from vital.environment import VitalEnvironment
from datetime import date, timedelta

client = Vital(
  api_key=os.environ["VITAL_API_KEY"],
  environment=VitalEnvironment.SANDBOX
)

# Fetch last 30 days of sleep data
end_date = date.today()
start_date = end_date - timedelta(days=30)

sleep_data = client.sleep.get(
  user_id=user.user_id,
  start_date=start_date.isoformat(),
  end_date=end_date.isoformat()
)

for sleep in sleep_data:
  print(f"Date: {sleep.date}")
  print(f"Duration: {sleep.duration_seconds / 3600:.1f} hours")
  print(f"Deep sleep: {sleep.deep_sleep_seconds / 3600:.1f} hours")
```

```javascript Node.js
import { VitalClient, VitalEnvironment } from '@tryvital/vital-node';

const client = new VitalClient({
  apiKey: process.env.VITAL_API_KEY,
  environment: VitalEnvironment.Sandbox,
});

// Fetch last 30 days of sleep data
const endDate = new Date().toISOString().split('T')[0];
const startDate = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000)
  .toISOString().split('T')[0];

const sleepData = await client.sleep.get(userId, {
  startDate,
  endDate,
});

sleepData.forEach(sleep => {
  console.log(`Date: ${sleep.date}`);
  console.log(`Duration: ${(sleep.duration_seconds / 3600).toFixed(1)} hours`);
});
```

```java Java
import com.vital.api.Vital;
import com.vital.api.core.Environment;
import com.vital.api.resources.sleep.requests.SleepGetRequest;
import java.time.LocalDate;

Vital vital = Vital.builder()
  .apiKey(System.getenv("VITAL_API_KEY"))
  .environment(Environment.SANDBOX)
  .build();

LocalDate endDate = LocalDate.now();
LocalDate startDate = endDate.minusDays(30);

SleepGetRequest request = SleepGetRequest.builder()
  .startDate(startDate.toString())
  .endDate(endDate.toString())
  .build();

var sleepData = vital.sleep().get(userId, request);
```

```go Go
import (
  "context"
  "time"

  vital "github.com/tryVital/vital-go"
  vitalclient "github.com/tryVital/vital-go/client"
)

// Calculate date range
endDate := time.Now().Format("2006-01-02")
startDate := time.Now().AddDate(0, 0, -30).Format("2006-01-02")

sleepData, err := client.Sleep.Get(
  context.TODO(),
  userId,
  &vital.SleepGetRequest{
    StartDate: startDate,
    EndDate:   &endDate,
  },
)
```

```swift Swift
let endDate = Date()
let startDate = Calendar.current.date(byAdding: .day, value: -30, to: endDate)!

let sleepData = try await VitalClient.shared.summary.sleep(
  userId,
  startDate,
  endDate
)
```
